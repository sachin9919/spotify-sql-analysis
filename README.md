# Spotify Data Analysis using SQL

## Overview
This project focuses on analyzing a Spotify dataset using SQL to extract meaningful insights about tracks, artists, and albums.

It covers the complete workflow from understanding the dataset structure to performing queries of varying complexity and optimizing them for better performance. The goal is to strengthen SQL skills while deriving useful insights from real-world data.

---

## Objectives
- Explore and understand the structure of the dataset  
- Perform data analysis using SQL queries  
- Solve business-oriented analytical problems  
- Apply advanced SQL concepts such as joins, subqueries, and window functions  
- Extract insights related to music trends and performance  

---

## Dataset Description
The dataset contains information related to:
- Artists and tracks  
- Albums and album types  
- Audio features such as danceability, energy, and tempo  
- Engagement metrics such as views, likes, comments, and streams  
- Platform-specific streaming data  

---

## Tools and Technologies
- SQL  
- PostgreSQL / MySQL  
- SQL Editor (pgAdmin or similar tools)  

---

## Analysis Performed
- Basic data retrieval and filtering  
- Aggregation and grouping operations  
- Comparative analysis across platforms  
- Ranking and top-performing tracks analysis  
- Advanced queries using subqueries and Common Table Expressions (CTEs)  

---

## Key Queries Covered
- Identifying tracks with high streaming counts  
- Finding top-performing artists and albums  
- Comparing streaming performance across platforms  
- Ranking tracks using window functions  
- Analyzing relationships between audio features  

---

## Key Insights
- A small number of tracks contribute significantly to total streams  
- Streaming performance varies across platforms  
- Certain artists consistently dominate top rankings
